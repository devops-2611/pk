# Helm Chart Commands Guide

## 1. Create Helm Chart

```bash
helm create hospital-mgmt
```

---

## 2. Validate Helm Chart

```bash
helm lint .
```

**Purpose:**
- Checks YAML syntax
- Checks Helm template errors
- Validates chart structure

---

## 3. Render Helm Templates (See Generated YAML)

```bash
helm template .
```

**Purpose:**
- Shows final Kubernetes YAML generated by Helm
- Does **NOT** deploy resources

---

## 4. Simulate Installation (Dry Run)

```bash
helm install two-tier . --dry-run
```

**Purpose:**
- Simulates installation
- Shows generated manifests
- Does **NOT** deploy resources

---

## 5. Install Helm Chart

```bash
helm install hospital-mgmt .
```

**Purpose:**
- Deploys the Helm chart to the Kubernetes cluster

---

## 6. Check Helm Releases

```bash
helm list
```

```bash
helm list --all
```

---

## 7. Upgrade Helm Release

```bash
helm upgrade hospital-mgmt .
```

**Purpose:**
- Updates the running application

---

## 8. Uninstall Helm Release

```bash
helm uninstall hospital-mgmt
```

**This removes:**
- Deployments
- Services
- ConfigMaps
- Secrets
- Other resources created by the chart
